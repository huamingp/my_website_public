实验3 IIC 通信

本章记录 STM32 IIC 通信配置、读写流程与调试方法。

原理​

IIC 采用双线串行总线结构，通过 SCL 提供时钟、SDA 传输数据，支持主从通信与多设备挂载。

起始与停止条件​

主机通过 SDA 在 SCL 高电平期间的电平变化产生起始与停止条件，从而定义一次传输的边界。

ACK 应答机制​

每字节传输后接收方需返回 ACK，若出现 NACK 通常表示地址错误、设备未就绪或总线异常。

代码​

工程中建议封装统一的读写接口，例如按“设备地址 + 寄存器地址 + 数据缓存”组织调用参数。

初始化流程​

初始化阶段应先配置 GPIO 复用开漏、开启 IIC 外设时钟，再设置时序寄存器满足目标速率。

读写流程​

写操作通常是“起始-地址-寄存器-数据-停止”，读操作一般为“写寄存器地址后重复起始再读数据”。

结果​

测试时可记录逻辑分析仪抓取波形、地址扫描结果与读写成功率，用于验证时序配置正确性。

现象记录​

若出现总线忙或通信超时，需优先检查上拉电阻、设备地址和时钟频率配置。

调试结论​

对于高噪声环境，适当降低总线速率并增加重试机制，通常可以显著提升稳定性。
